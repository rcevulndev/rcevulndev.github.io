Fuji Electric V-SFT and Tellus Lite V-Simulator X1 File Parsing Use-After-Free VulnerabilityAWE-2021-055

CVE Number -
Vendor Fuji Electric
Product V-SFT and Tellus Lite V-Simulator
Affected version ≤ v4.0.11.0 (Tellus Lite V-Simulator )
≤ v6.1.1.0 (V-SFT)
Patched version v4.0.12.0 (Tellus Lite V-Simulator)
v6.1.3.0 (V-SFT)
Summary A use-after-free vulnerability exists that could result in loss of data or execution of arbitrary code due to lack of input validation, when a malicious X1 file is opened.
Resources ICS-CERT/CISA Advisory
Timeline 13.04.2021: Report sent to JPCERT/CC
13.04.2021: Received tracking number from JPCERT/CC
27.04.2021: Asking JPCERT/CC for a status update
28.04.2021: JPCERT/CC informs me that they are still waiting for a reply from the vendor
19.05.2021: Asking JPCERT/CC for a status update
19.05.2021: Received feedback from JPCERT/CC that the vendor plans to tentatively address the issues in October 2021
10.06.2021: Asking JPCERT/CC for a status update
10.06.2021: Received feedback that the tentative schedule for public disclosure remains October, after patches have been made available by the vendor
01.10.2021: Asking JPCERT/CC for a status update
02.10.2021: JPCERT/CC informs me that they will contact the vendor to ask for an update
26.10.2021: Asking for a status update, pointing out that ICS-CERT has published advisory icsa-21-299-01
27.10.2021: JPCERT/CC explains that Fuji Electric received the same vulnerabilities via ICS-CERT, which in turn received the reports from ZDI, and that Fuji Electric forgot to update JPCERT/CC. JPCERT/CC further states that they contacted Fuji Electric multiple times to identify which of the CVEs are assigned for the specific vulnerabilities that I discovered and reported, but that Fuji Electric has been unresponsive. Moreover, Fuji Electric did not inform JPCERT/CC when ICSA-21-299-01 from ICS-CERT was scheduled to be published.
