Schneider Electric EcoStruxure Power Build Rapsody SSD File Parsing Heap-based Buffer Overflow VulnerabilityAWE-2025-006

CVE ID CVE-2024-11139
Vendor Schneider Electric
Product EcoStruxure Power Build Rapsody
Affected versions ≤ v2.5.2 NL
≤ v2.7.1 FR
≤ v2.7.5 ES
≤ v2.5.4 INT
Patched versions v2.7.2 NL
v2.7.12 FR
v2.7.52 ES
N/A for INT
Summary A heap-based buffer overflow vulnerability exists in EcoStruxure Power Build Rapsody when parsing SSD files. The issue can potentially be exploited to execute arbitrary code, but requires a legitimate user to open a malicious SSD file.
Resources ICSA-25-023-05SEVD-2025-014-09
Timeline (abridged) 27.11.2023: Report with five vulnerabilities sent to CISA.
18.12.2023: Vendor states that they reviewed one issue, a stack-based buffer overflow, and cannot reproduce it.
21.12.2023: Vendor states that two of the reported issues were reviewed and are false positives. Moreover, they state that the other issues cannot be assessed by them, because "only hexadecimal addresses were provided".
Note: Vendor is referring to the debugging and disassembly listings included in the report (which also had the proof-of-concept files attached to trigger each highlighted issue) that was shared with them.
09.01.2024: Vendor states that the issues cannot be reproduced. Vendor further states that if the issues could be reproduced, they would have low occurrence and criticality scores, and "would not do any damage to existing projects, existing rapsody installation or the OS". Moreover, they state that "no remote code that could be executed because DEP (Data Execution Prevention) is enabled".
19.01.2024: Vendor asked their internal pentest team to verify the issues I reported.
05.02.2024: Vendor states that their pentest team was able to confirm all reported vulnerabilities.
07.02.2024: Vendor states that from the CVSS properties for Confidentiality, Integrity, and Availability, the memory corruption issues only have a low impact on Confidentiality, but that Integrity and Availability are not affected.
22.02.2024: Vendor states that crafting a ROP chain requires significant tooling and effort.
28.02.2024: Vendor states that the issues will be fixed, but due to the low CVSS score, no CVE will be assigned.
05.03.2024: Vendor states they did an internal reevaluation, and revised the CVSS score, and are going to assign one CVE.
26.09.2024: Vendor states that all five reported vulnerabilities have the same root cause, and therefore only assign one CVE.
14.01.2025: Public release of security advisory
